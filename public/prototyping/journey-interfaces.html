Journey

Interfaces

This journey shows how Python grows from simple functions to callable APIs, object interfaces, protocols, and metaclasses.

Start with functions as named behavior.

Functions are the first abstraction boundary because they name behavior and control how callers provide information.

  • Functions

    package behavior behind a name

  • Keyword-only Arguments

    make important call-site choices explicit

  • Positional-only Parameters

    hide parameter names that should remain implementation details

  • Args and Kwargs

    accept flexible call shapes when forwarding or adapting APIs

  • Multiple Return Values

    return multiple related values as a tuple

Use functions as values.

Python functions can capture state, be passed around, and wrap other functions.

  • Global and Nonlocal

    control where assignment happens

  • Closures

    capture state in nested functions

  • Recursion

    solve self-similar problems with a base case

  • Lambdas

    write small unnamed functions for expression positions

  • Decorators

    wrap behavior without changing call sites

  • Partial Functions

    show how to pre-fill arguments with `functools.partial`

Bundle behavior with state.

Classes become useful when data and behavior need to move together behind a stable interface.

  • Classes

    bundle state and behavior into a new object type

  • Inheritance and Super

    reuse and extend behavior through parent classes

  • Dataclasses

    generate common methods for data containers

  • Properties

    keep attribute syntax while adding computation or validation

  • Special Methods

    connect objects to Python syntax and built-ins

  • Truth and Size

    make objects work with truth tests and `len()`

  • Container Protocols

    support membership, lookup, and assignment syntax

  • Callable Objects

    make stateful instances callable like functions

  • Operator Overloading

    define operators only when the operation is unsurprising

  • Attribute Access

    customize fallback lookup and assignment carefully

  • Descriptors

    explain the protocol behind methods, properties, and managed attributes

  • Metaclasses

    customize class creation when ordinary class tools are not enough
